Create a comfortable and reliable PowerShell script for migrating data from Microsoft SQL Server to MariaDB. A CSV file containing the table and field mapping information already exists and must be used as the basis for the migration. The key challenge is that identity values, primary keys, foreign keys, tenant IDs, and GUIDs must not simply be copied 1:1 from SQL Server to MariaDB. Instead, new IDs and GUIDs must be generated in the MariaDB target database, especially for tenants, and these newly generated values must then be consistently mapped and propagated to all related tables so that the migrated data remains fully relationally valid and operational. Develop a clear migration strategy and implement it in the PowerShell script so that all dependencies, relationships, and references are preserved correctly in the MariaDB target database. The solution must also take into account that additional tenants will be added to the target database in the future, so the migration must not create conflicts with existing or future data. You may create a copy of the target database before running the migration, execute the script against this copy, and repeatedly test and refine the process until the result is correct, consistent, and ready for use.
